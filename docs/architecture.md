# Архитектура проекта

Проект представляет собой браузерную игру, написанную на Vanilla JS, с использованием объектно-ориентированного подхода. Архитектура разделена на логику движка, пользовательский интерфейс (UI) и искусственный интеллект (AI).

## Взаимодействие UI и движка

- **`chess-engine.js` (Логика)**: Инкапсулирует правила шахмат и механику предметов. Не зависит от DOM. Класс `ChessEngine` отвечает за генерацию ходов, проверку легальности и применение ходов (`executeMoveAndUpdate`).
- **`app.js` (Представление и Контроллер)**: Отвечает за отрисовку состояния доски в DOM, обработку событий мыши/касаний (Drag & Drop), и управление переходами между экранами. Экземпляр `ChessEngine` хранится в `app.js` и вызывается при действиях игрока.
- **`chess-ai.js` (ИИ)**: Содержит класс `ChessAI`, который использует метод `clone()` движка для симуляции ходов и алгоритм минимакс/альфа-бета отсечения.
- **`items-db.js` и `locales.js`**: Статические базы данных для предметов и локализации соответственно.

```mermaid

    UI[app.js (UI & Events)] -->|Вызов методов| Engine[chess-engine.js (Game Rules)]
    UI -->|Получает ходы| AI[chess-ai.js (Minimax AI)]
    AI -->|Клонирует состояние| Engine
    Engine -->|Запрашивает данные предметов| Items[items-db.js]
    Engine -->|Использует сущности| Piece[piece-entity.js]
    UI -->|Берет переводы| Locales[locales.js]
```

## Игровой цикл

Игровой цикл (Game Loop) управляется через события (event-driven), а не постоянный цикл обновления (как в `requestAnimationFrame`), так как игра пошаговая.

1. **Ожидание ввода**: `app.js` ждёт действия пользователя (клик или Drag & Drop).
2. **Обработка хода**: 
   - Пользователь перемещает фигуру.
   - `app.js` запрашивает у `engine.generateLegalMoves()` список доступных ходов.
   - Если ход валиден, вызывается `engine.executeMoveAndUpdate(move)`.
3. **Обновление UI**: 
   - Вызывается функция рендера доски `renderBoard()`.
   - Проигрываются звуки, обновляется золото и статистика.
4. **Ход ИИ**:
   - Если следующий ход за ИИ, `app.js` вызывает метод ИИ с небольшой задержкой (через `setTimeout`), чтобы не блокировать UI.
   - ИИ вычисляет лучший ход и возвращает его в `app.js`.
   - Цикл возвращается к шагу 2.

```javascript
// Пример базового игрового цикла в app.js (псевдокод)
async function handleMove(move) {
    const result = engine.executeMoveAndUpdate(move);
    updateUI(result);
    
    if (engine.currentTurn === 'black' && !engine.gameOver) {
        showThinkingIndicator();
        setTimeout(() => {
            const bestMove = ai.calculateBestMove(engine);
            handleMove(bestMove);
        }, 100);
    }
}
```
